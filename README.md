# Batmanuel

**Batmanuel** is a code quality platform built with NestJS. It analyzes repositories to measure code quality, detect security vulnerabilities, flag outdated or vulnerable dependencies, and identify duplicated code. It exposes an API consumed by a CLI and integrates natively with CI/CD pipelines (GitHub Actions initially).

## Architecture Overview

Batmanuel acts as an orchestrator rather than a single monolithic analyzer. It:

1. Receives analysis reports generated by external tools (linters, SAST scanners, dependency scanners).
2. Normalizes those reports into a common `AnalysisReport` model.
3. Calculates a consolidated quality/security score per project, branch, and commit.
4. Exposes that data through a REST API, consumed by a CLI, CI/CD pipelines, and an Angular dashboard.

```
CLI / GitHub Actions  --->  Batmanuel API (NestJS)  --->  Normalized report + score
                                   |
                                   v
                            PostgreSQL storage
                                   |
                                   v
                            Angular dashboard
```

## Getting Started

### Prerequisites

- Node.js 18+
- npm

### Installation

```bash
npm install
```

### Running locally

```bash
npm run start:dev
```

The API will be available at `http://localhost:3000`.

### Testing a request

```bash
curl -X POST http://localhost:3000/analyze \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <YOUR_TOKEN>" \
  -d '{"projectId":"my-project-backend","branch":"main","commit":"abc123"}'
```

## API Routes

### POST /auth/token

Generates an access token for a given project. This token must be used to authenticate all subsequent requests to protected routes.

**Request body:**

```json
{
  "projectId": "my-project-backend"
}
```

**Response:**

```json
{
  "projectId": "my-project-backend",
  "token": "<GENERATED_TOKEN>",
  "expiresIn": "30d"
}
```

---

### POST /analyze

Receives project/branch/commit information, runs the analysis engines, and returns a consolidated report. Requires authentication (see [Authentication](#authentication)).

**Request body:**

```json
{
  "projectId": "my-project-backend",
  "branch": "main",
  "commit": "abc123"
}
```

**Response:**

```json
{
  "projectId": "my-project-backend",
  "branch": "main",
  "commit": "abc123",
  "timestamp": "2026-07-15T15:00:00.000Z",
  "score": 82,
  "metrics": {
    "securityCritical": 1,
    "securityHigh": 2,
    "securityMedium": 5,
    "qualitySmells": 12,
    "duplications": 3,
    "outdatedDeps": 2
  },
  "issues": [
    {
      "engine": "semgrep",
      "type": "security",
      "severity": "critical",
      "ruleId": "sql-injection",
      "message": "Possible SQL injection",
      "file": "src/users/users.service.ts",
      "line": 42
    }
  ]
}
```

| Field                                | Type   | Description                                    |
| ------------------------------------ | ------ | ---------------------------------------------- |
| score                                | number | Overall quality/security score (0-100)         |
| metrics.securityCritical/High/Medium | number | Count of security issues by severity           |
| metrics.qualitySmells                | number | Count of code smells detected                  |
| metrics.duplications                 | number | Count of duplicated code blocks                |
| metrics.outdatedDeps                 | number | Count of outdated/vulnerable dependencies      |
| issues[].type                        | string | `security`, `quality`, or `dependency`         |
| issues[].severity                    | string | `critical`, `high`, `medium`, `low`, or `info` |

---

### GET /projects/:id/summary

Returns a historical summary for a given project, including score trend across recent analyses. Requires authentication.

**Response:**

```json
{
  "projectId": "my-project-backend",
  "lastScore": 82,
  "trend": [76, 78, 80, 82],
  "lastAnalysisAt": "2026-07-15T15:00:00.000Z"
}
```

## Authentication

All routes except `POST /auth/token` require a Bearer token in the `Authorization` header:

```
Authorization: Bearer <YOUR_TOKEN>
```

The `TokenGuard` (in `src/auth/token.guard.ts`) validates this header and scopes each request to the corresponding project.

## Swagger / API Documentation

Interactive API documentation is powered by `@nestjs/swagger` and is automatically enabled in the **development** environment.

### Accessing the docs

```
http://localhost:3000/docs
```

On a deployed develop instance, replace `localhost:3000` with the instance's public URL, e.g.:

```
https://batmanuel-develop.your-domain.com/docs
```

### Authenticating in Swagger UI

1. Open `/docs`.
2. Click **Authorize**.
3. Paste your Bearer token.
4. All protected routes (`/analyze`, `/projects/:id/summary`) can now be tested directly from the UI.

### Running with the develop environment

```bash
NODE_ENV=develop npm run start:dev
```

Or via `.env`:

```
NODE_ENV=develop
PORT=3000
```

## GitHub Actions Integration

A sample workflow is provided at `.github/workflows/code-quality.yml`. It:

1. Checks out the repository.
2. Builds a report payload with project, branch, and commit information.
3. Sends the payload to `POST /analyze`, authenticating with a Bearer token.
4. Reads the returned `score` and fails the pipeline if it falls below a defined threshold.

**Required repository secrets:**

| Secret              | Description                                |
| ------------------- | ------------------------------------------ |
| `QUALITY_API_URL`   | Public URL of the Batmanuel API            |
| `QUALITY_API_TOKEN` | Bearer token used to authenticate requests |

**Example step:**

```yaml
- name: Send report to Batmanuel
  run: |
    curl -s -X POST "$QUALITY_API_URL/analyze" \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer $QUALITY_API_TOKEN" \
      -d @report.json \
      -o response.json
  env:
    QUALITY_API_URL: ${{ secrets.QUALITY_API_URL }}
    QUALITY_API_TOKEN: ${{ secrets.QUALITY_API_TOKEN }}
```

## Roadmap

- **Persistence**: store historical reports in PostgreSQL for long-term trend analysis.
- **Dashboard**: expand the Angular frontend consuming `GET /projects/:id/summary` and related endpoints.
- **CLI**: ship a standalone CLI (`npx batmanuel analyze`) that collects local reports and pushes them to the API, simplifying pipeline usage.
- **Additional pipelines**: extend integration beyond GitHub Actions to GitLab CI and Bitbucket Pipelines.
